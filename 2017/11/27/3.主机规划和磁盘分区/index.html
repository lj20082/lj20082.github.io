Linux与硬件的搭配

  1. 各个组件或设备在Linux下面都是一个文件（一切皆文件）

  2. 硬件配置思考角度：

    • 游戏机/工作机的考虑
    • 性价比的考虑（主流级产品、省电）
    • 支持性的考虑

  3. 各硬件设备在Linux中的文件名

设 备 设备在Linux内的文件名
IDE硬盘 /dev/hd[a-d]
SCSI/SATA/USB硬盘 /dev/sd[a-p]
U盘 /dev/sd[a-p] 与SATA相同
软驱 /dev/fd[0-1]
打印机 25针：/dev/lp[0-2] USB:/dev/usb/lp[0-15]
鼠标 USB:/dev/usb/mouse[0-15] PS2:/dev/psaux
当前CD ROM/DVD ROM /dev/cdrom
当前鼠标 /dev/mouse

磁盘分区

  1. 磁盘连接方式与设备文件名的关系

    • IDE设备可以被区分为主设备和从设备

      | IDE | Master | Slave |
      | :————-: | :——: | :——: |
      | IDE(Primary) | /dev/hda | /dev/hdb |
      | IDE2(Secondary) | /dev/hdc | /dev/hdd |

    • SATA/USB/SCSI都是SCSI模块驱动的，Linux根据Linux内核检测到磁盘的顺序决定设备名

  2. 磁盘的组成复习

    • 磁盘的组成（盘片、机械手臂、磁头、主轴马达），数据的写入在盘片上面
    • 盘片上面又细分出扇区(Sector)和柱面(Cylinder)2种单位，其中每个扇区512Byte大
    • 整个磁盘的第一个扇区特别重要，它记录了整块磁盘的重要信息，重要记录了：
      • 主引导分区（Master Boot Record,MBR）:可以安装引导加载程序的地方，446byte
      • 分区表（partition table）:记录整块磁盘分区的状态，有64byte

  3. 磁盘分区表

    • 分区只是针对64byte的分区表进行设置

    • 磁盘默认的分区表仅能写入4组分区信息，

    • 这4组分区信息我们称为主(Primary)或扩展(Extended)分区

    • 分区的最小单位为柱面，利用参考柱面号码的方式，在分区表64byte容量中，总共分为4组记录区，每组记录区记录了该区段的起始和结束的柱面号码，设备名分别为/dev/hda1、/dev/hda2、/dev/hda3、/dev/hda4

    • 当系统写入磁盘时，会参考磁盘分区表，才能对整个分区进行数据的处理

    • 分区优点：数据的安全性，系统性能的考虑

    • 扩展分区的目的是使用额外的扇区来记录分区信息，扩展分区本身不能被拿来格式化

    • 由扩展分区继续切出来的分区，被称为逻辑分区，设备文件名从/dev/hda5开始

    • 逻辑分区的设备名称号码从5号开始

    • 关于主分区、扩展分区。逻辑分区的特性的简单定义

      • 主分区和控制分区总共最多可以有4个（硬盘的限制）
      • 扩展分区最多只能有1个（操作系统的限制）
      • 逻辑分区是扩展分区持续切割出来的分区
      • 能格式化后作为数据访问的分区是主分区和逻辑分区，扩展分区无法格式化
      • 逻辑分区数量依操作系统不同，在Linux中，IDE硬盘最多有59个逻辑分区（5-63号），SATA硬盘最多有11个逻辑分区（5-15号）
      • 如果扩展分区被破坏，所有逻辑分区将会被删除

    • 开机流程和主引导分区

      • 开机流程到操作系统之前的动作：

        • BIOS:开机主动执行的韧体（写入到硬件上的一个软件程序），识别第一个可开机的设备

        • MBR:第一个可开机设备的第一个扇区内的主引导分区块，内包含引导加载程序

        • 引导加载程序(Boot Loader)：一支可读取内核文件来执行的软件

        • 内核文件：开始操作系统的功能

          由上可知，BIOS和MBR都是硬件本身会支持的功能，至于Boot Loader则是系统安装在MBR上的一套软件。由于MBR仅有446bytes，因此这个引导加载程序是非常小而完美的。

      • CMOS是记录各项硬件参数且嵌入到主板上面的存储器，BIOS就是在开机时候计算机系统主动执行的第一个程序，BIOS依据用户设置取得能开机的硬盘，并且到该硬盘里面去读取第一个扇区MBR位置。MBR这个仅有446bytes的磁盘容量里放置最基本的引导加载程序，这个引导加载程序的目的是再加载内核文件。

      • Boot Loader任务

        • 提供菜单 : 用户选择不同的开机选项，多重引导的重要功能
        • 载入内核程序 : 直接指向可开机的程序区段来开始操作系统
        • 转交其他loader : 将引导加载功能转交给其他loader负责

      • 多重引导

        • 每个分区都有自己的启动扇区
        • 图中的系统分区为第一及第二分区
        • loader只会认识自己的系统分区内的可开机内核文件，以及其它loader
        • loader可直接指向或者间接将管理权转交给另一个管理程序

      • 如果要安装多重引导，最好先安装windows再安装Linux。（Linux安装可以选择引导加载程序的安装位置，可手动设置菜单。windows会主动覆盖原引导加载程序）

  4. Linux安装模式下磁盘分区的选择

    • 目录树结构：整个目录树结构最重要的就是那个根目录，整个根目录的表示方法就是一条斜线”/“
    • 文件系统与目录树的关系-挂载，所谓挂载就是利用一个目录当成进入点，将磁盘分区的数据放置在该目录下，也就是说进入该目录就可以读取该分区，这个操作我们称为挂载，那个进入点的目录我们称为挂载点
