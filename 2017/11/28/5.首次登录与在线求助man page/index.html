重要的热键【Tab】【Ctrl】- c,【Ctrl】-d

  • 【Tab】
    • Tab 在UNIX Like的Shell中，具有命令补全和文件补全的功能，避免我们打错命令和文件名
    • Tab接在一串命令的第一个命令后面，则为命令补全
    • Tab接在一串命令的第二个命令以后时，则为文件补全
  • 【Ctrl】-c ：中断目前程序
  • 【Ctrl】-d :键盘键入结束EOF,也可替代exit,如想要直接离开文字界面，按下ctrl-d就行了

man page(UNIX LIKE)

  • man是manual(操作说明)简写，通过 man xxx可查看各命令操作说明,呼出的页面我们称为man page
  • whatis xx = man -f xx //查找跟xx命令相关的说明文件
  • apropos xx = man -k xx //在系统的说明文件中，只要有xx就列出来

info page(LINUX)

  • info命令查看命令说明，info page是将文件数据拆成一个个的段落，每个段落用自己的页面撰写，并且各页面还有跳转链接，每个独立的页面称为一个节点
  • 在命令行界面，任何不知道的命令或文件格式，使用man或info查询
  • 如果想架设一些其他服务或利用一整套软件达成某项功能时，查看/usr/share/doc下面是否有该服务的说明文档

正确关机方法

数据同步写入磁盘sync

Linux系统中，为了加快数据的读取速度，默认情况下某些已经加载内存中的数据将不会直接写回硬盘，而是先暂存在内存当中，这样如果一个数据重复改写，某些在内存中的数据由于还没有写入硬盘，速度上快很多，但万一系统因为某些特殊情况不正常关机，由于数据未被写入硬盘中，会造成数据的更新不正常。使用sync能将内存中尚未更新的数据写入硬盘中。shutdown/reboot/halt等命令均已在关机前进行了sync工具的调用。

惯用的关机命令：shutdown

shutdown可以达成以下工作：

  • 可以自由选择关机模式：是要关机、重启或进入单用户操作模式均可

  • 可以设置关机时间：可以设置现在立即关机，也可以设置某一个特定的时间才关机

  • 可以自定义关机消息：在关机之前，可以将自己设置的消息传送给在线用户

  • 可以仅发出警告消息：有时可能你要进行一些测试，而不想让其他用户干扰，或者是明白的告诉用户某段时间要注意一下，这个时候可以用shutdown来通知用户，但却不是真的要关机

  • 可以选择是否要用fask检查文件系统

    需要注意的是，时间参数务必加入命令中，否则shutdown会自动跳到run-level 1(就是单用户维护的登录情况)

开机过程问题排查

  • 磁盘坏道或文件系统发生错误（数据损坏）

    原因：1.断电或不正常关机所导致的文件系统发生错误 2.硬盘使用率过高或主机所在环境不良

    解决方法：

    1. 如果根目录没有损坏，Ctrl+D单用户root账户维护，执行命令fsck /dev/sda7(fsck为文件系统检查命令，/dev/sda7为错误的分区，按实际情况填入)，这时屏幕开始修复磁盘，修复完成后，reboot重启。
    2. 如果根目录损坏，需要将硬盘拔掉，到另外一台Linux系统的计算机上，不挂载（mount）该硬盘，然后以root身份执行fsck /dev/sdb1(/dev/sdb1指的是硬盘设备文件名，按实际情况填入)
    3. 如果硬盘整个坏掉，救出能救的数据，换新硬盘

    硬盘维护：

    1. 妥善保管硬盘
    2. 划分不同分区（建议 / 、/boot 、/usr 、/home 、/var）

  • 忘记root密码
