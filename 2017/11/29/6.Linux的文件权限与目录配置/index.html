Linux的文件权限与目录配置

发表于 2017-11-29 | 分类于 linux | | 阅读次数:
字数统计: 2,623 | 阅读时长 ≈ 10
Linux一般将文件可存取访问的身份分为3个类别，分别是owner、group、others，且三种身份各有read、write、execute等权限。本文主要讲述用户和用户组、Linux文件权限的重要性、如何改变文件权限与属性、目录与文件权限、目录树的相关知识...

用户与用户组

  • 文件所有者

    Linux是个多用户、多任务的系统，为了考虑个人隐私及每个人喜好的工作环境，所以有文件所有者概念。通过设置权限，实现文件权限的合理分配。

  • 用户组概念

    用户：个人

    用户组：多个个人组成的集合

    用户和用户组可以多对多，有了用户和用户组，实现了更方便的分配权限。

  • 其他人的概念

    非用户或用户组的其它人，Linux中任何一个文件都具有User/Group/Others 3种身份的个别权限

  • root：万能的天神

    可以到达任何地方，拥有最高权限

  • Linux用户身份与用户组记录的文件

    默认情况下所有系统上的账号和一般身份用户，还有root的相关信息，都是记录在/etc/passwd这个文件内，至于个人的密码则记录在/etc/shadow这个文件下。Linux所有的组名都记录在/etc/group内，这3个文件是Linux系统里面账号、密码、用户组信息的集中地。

Linux文件权限概念

ls -al显示目录下的所有文件名与相关属性，而参数-al则表示列出所有文件详细的权限与属性（包括隐藏文件，就是第一个字符是’.’的文件）。

第一列待办这个文件的类型与权限，

  • 若是[d]则是目录
  • 若是[-]则是文件
  • 若是[l]则表示连接文件（linkfile）
  • 若是[b]则表示设备文件里面的可供存储的接口设备
  • 若是[c]则表示设备文件里面的串行端口设备，如键盘、鼠标

接下来的字符中，以3个为一组，且均为“rwx”的3个参数的组合，其中r代表可读(read)，w代表可写(write)，x代表执行(execute)，如果没有权限，就会出现减号[-]而已（即3个权限的位置固定）

  • 第一组为 文件所有者的权限
  • 第二组为 同用户组的权限
  • 第三组为其他非本用户组的权限

第二列表示有多少文件名连接到此节点

每个文件都会将其属性和权限记录到文件系统的i-node中，我们使用的目录树却是使用文件名来记录，因此每个文件名都会连接到一个i-node，这个属性记录的就是有多少不同的文件名连接到相同的一个i-node号码

第三列表示这个文件（或目录）的所有者账号

第四列表示这个文件的所属用户组

Linux系统下，一个账号会附属一个或多个用户组

第五列为这个文件的容量大小，默认单位为B

第六列为这个文件的创建日期或最近的修改日期

使用ls -l --full-time显示完整的时间格式

使用LANG=en_US修改语言

第七列为文件名，如果文件名前面多一个.则代表这个文件为隐藏文件

Linux文件权限的重要性

  • 数据安全性（系统保护）
  • 团队开发软件或数据共享的功能
  • 未将权限设置妥当会造成很多危害

如何改变文件属性与权限

cp复制文件给其他人时，因为复制行为会复制执行者的权限与属性，所以必须修改这个文件的所有者或用户组。

使用chmod设置权限的2种方法，分别可以使用数字或者符号来进行权限的更改。

数字类型改变文件权限

Linux文件的基本权限9种，分别是owner/group/others三种身份各有自己的read/write/execute权限，文件的权限字符为“-rwxrwxrwx”，9个权限3个3个一组，各权限的分手对照表如下，r:4 w:2 x:1，每种身份的3个权限分数是需要累加的。

eg. -rwxrwx— 分数则是 owner = 4+2+1 = 7，group = 4+2+1 = 7，others = 0+0+0 = 0，所以该文件的权限数字就是770，

符号类型改变文件权限

user/group/others 3种身份使用u/g/o 表示这3种身份的权限，此外a代表全部的身份，读写操作的权限写作rwx，

命令名 身份 操作 权限 文件或目录名
chmod u/g/o/a +加入 -除去 =设置 r/w/x 文件或目录

示例：

chmod u=rwx，go=rx .bashrc 设置文件.bashrc权限为 -rwxr-xr-x

chmod a+w .bashrc 不知道原先文件属性，只是增加这个文件每个人都可以写入的权限

chmod a-x .bashrc 去掉所有人的可执行权限

目录或文件的权限意义

  • 权限对文件的重要性

    文件是实际含有数据的地方，包括一般文本文件、数据库内容文件、二进制可执行文件等，因此权限对于文件来说：

    • r(read) : 可读取文件的实际内容
    • w(write) : 可编辑、新增或者是修改该文件的内容（但不含删除该文件）
    • x(execute) : 该文件具有可以被系统执行的权限

    文件的权限主要是针对文件的内容而言，因此w权限只有写入、编辑、修改文件的内容权限，不具备删除该文件的权限

  • 权限对目录的重要性

    文件是存放实际数据的所在，目录主要的内容是记录文件名列表，文件名与目录有强烈的关联，所以权限对于目录来说：

    • r(read) : 表示具有读取目录结构列表的权限，可以查询该目录下的文件名数据，可以使用ls命令将目录内容列表展示
    • w(write) : 表示你具有更改该目录结构列表的权限：新建新的文件和目录、删除已经存在的文件和目录（不论文件权限）、将已存在的文件或目录重命名、转移该目录内的文件、目录位置
      x (access directory) : 表示用户能否进入该目录成为工作目录的用途，工作目录是指你所在的主文件夹，变换目录的命令是cd

    工作目录对应命令的执行非常重要，如果你在某目录下不具有x的权限，那么你无法切换到该目录下，也就无法执行该目录下的任何命令，即使你具有该目录的r权限。要开放目录给任何人浏览时，应该至少给予r及x的权限，不然用户只能看到文件名，无法查阅文件内容（权限不足）

Linux文件种类和扩展名

文件种类

  1. 普通文件【 - 】

    • 纯文本文件
    • 二进制文件
    • 数据格式文件

  2. 目录文件 【d】

    • 第一个属性为 d

  3. 连接文件（link）

    • 类似windows下面的快捷方式，第一个属性【l】

  4. 设备与设备文件
    与系统外设及存储相关的一些文件，通常集中在/dev目录，通常分为块设备文件和字符设备文件

    • 块设备文件 ：存储数据，提供系统随机访问的接口设备，如硬盘、软盘等，第一个属性是【b】
    • 字符设备文件 ：一些串行接口的接口设备，如键盘、鼠标等，第一个属性是【c】

  5. 套接字

    • 数据接口文件，这种文件常用在网络上的数据连接，第一个属性是【s】，通常/var/run目录中能看到这种文件

  6. 管道

    • FIFO也是一个特殊的文件类型，主要目的是解决多个程序同时访问一个文件所造成的错误问题，第一个属性 【p】

Linux文件扩展名

Linux文件没有所谓的扩展名，一个Linux文件能不能被执行，与它的第一列的10个属性有关，与文件名扩展名无关。x代表可以被执行，但执行是否成功与文件内容有关。由于我们希望通过扩展名了解该文件是什么东西，所以我们还是会以适当的扩展名表示文件种类，常用扩展名如下：

  • *.sh 脚本或批处理文件
  • Z / .tar / .tar.gz / .zip / *.tgz ：经过打包的压缩文件
  • .html / .php ： 网页相关文件

Linux文件长度限制（Ext2/Ext3文件系统）

  • 单一文件或目录的最大容许文件名为255个字符
  • 包含完整路径名称及目录(/) 的完整文件名为4096个字符

Linux文件名的限制

  • 避免特殊字符，如 *?<>:;&[]{}()|\’””`等
  • 避免文件名开头 - 或 + 命名
